## One-Time Setup: Add the Private Key Secret

### Step 1 — Get the Private Key

1. Go to: **GitHub → Settings (top-right avatar) → Developer settings → GitHub Apps**
2. Find **"Infinity Orchestrator"** (or create it if it doesn't exist)
3. Click **"Generate a private key"**
4. Download the `.pem` file

> If you no longer have access to the private key, you can regenerate one on
> the GitHub App settings page without changing the App ID or re-installing it.

### Step 2 — Add the Secret to This Repo

1. Go to: **https://github.com/InfinityXOneSystems/XPS_INTELLIGENCE_SYSTEM/settings/secrets/actions**
2. Click **"New repository secret"**
3. Name: `GH_APP_PRIVATE_KEY`
4. Value: paste the full contents of the `.pem` file (including `-----BEGIN RSA PRIVATE KEY-----` header/footer)
5. Click **"Add secret"**

> **Optionally**, add it as an **organisation secret** at
> `https://github.com/organizations/InfinityXOneSystems/settings/secrets/actions`
> so all repos in the org share it automatically.

## Triggering the Auto-Merge Workflow

Once the `GH_APP_PRIVATE_KEY` secret is set (or even without it, using `GITHUB_TOKEN`):

### Via GitHub UI

1. Go to: **Actions → 🔀 Auto-Merge & Deploy to Railway**
2. Click **"Run workflow"**
3. Enter PR number (default: `172`)
4. Select merge method (`squash` recommended)
5. Click **"Run workflow"**

### Via GitHub API

```bash
curl -X POST \
  -H "Authorization: token YOUR_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  "https://api.github.com/repos/InfinityXOneSystems/XPS_INTELLIGENCE_SYSTEM/actions/workflows/auto_merge_deploy.yml/dispatches" \
  -d '{"ref":"main","inputs":{"pr_number":"172","merge_method":"squash","deploy_after_merge":"true"}}'
```

### Via `gh` CLI

```bash
gh workflow run auto_merge_deploy.yml \
  --repo InfinityXOneSystems/XPS_INTELLIGENCE_SYSTEM \
  --ref main \
  --field pr_number=172 \
  --field merge_method=squash \
  --field deploy_after_merge=true
```
